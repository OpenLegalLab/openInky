# Implementation Plan

[Overview]
Develop a production-ready Microsoft Word Add-in with a React-based sidebar AI assistant that integrates with a custom LLM endpoint to provide context-aware text improvements using native Track Changes.

The system will use the Microsoft standard `yo office` scaffolding (React + Fluent UI) for native integration, and will securely connect to a custom LLM endpoint (default: `llm.lauxlawyers.ch`). By utilizing `jsdiff`, the add-in will compute minimal word-level differences between original and AI-generated text. These differences will be inserted into the document with Track Changes enabled, providing clean, human-like redlining without blindly replacing entire paragraphs.

[Types]
Introduce robust TypeScript types for LLM requests, AI diff mapping, and configuration state.

- `LLMConfig`: Interface defining the LLM endpoint configuration (baseUrl, apiKey, model).
- `LLMMessage`: Interface for system and user messages (role, content).
- `LLMRequest` / `LLMResponse`: Interfaces corresponding to the OpenAI chat completions schema.
- `DiffChunk`: Interface defining the outcome of `jsdiff` (value: string, added?: boolean, removed?: boolean) to map changes to Word Ranges.

[Files]
Scaffold the base project using `yo office` and add key modules for LLM API integration and Word diff logic.

- New files to be created:
  - `src/taskpane/components/App.tsx`: Main React application component containing the UI (Prompt box, action buttons, response preview).
  - `src/taskpane/components/Settings.tsx`: Configuration UI for API key and model selection.
  - `src/services/llmService.ts`: Handles requests to the LLM API endpoint with retry logic and error handling.
  - `src/services/diffService.ts`: Wraps `jsdiff` to compute differences between strings.
  - `src/services/wordService.ts`: Core Word interaction logic (reading text, enabling Track Changes, applying diffs).
- Existing files to be modified:
  - `manifest.xml`: Update to define Add-in metadata, permissions (ReadWriteDocument), and host configurations.
  - `package.json`: Add dependencies (`diff` for jsdiff).
- Files to be deleted or moved:
  - Default placeholder components generated by `yo office` that are not needed.

[Functions]
Implement core service functions for LLM interaction and diff mapping.

- New functions:
  - `streamCompletion(config: LLMConfig, messages: LLMMessage[])` in `llmService.ts`: Sends data to the configurable endpoint.
  - `computeWordDiff(original: string, modified: string)` in `diffService.ts`: Returns an array of `DiffChunk` using `jsdiff.diffWords`.
  - `applyTrackedChanges(range: Word.Range, diffs: DiffChunk[])` in `wordService.ts`: Iterates over diffs. Enables Track Changes (`trackAll`), then applies insertions and deletions sequentially to the document range.
  - `getSelectedTextOrDocument()` in `wordService.ts`: Retrieves the current user selection or full document body if selection is empty.

[Classes]
No heavy OOP classes required; will rely on React Functional Components and modular service objects.

- New classes:
  - `WordIntegrationService`: (Optional class, though module exports are preferred) to encapsulate Office.js context management.

[Dependencies]
Include necessary external libraries while keeping the payload light.

- `diff`: (`jsdiff`) for word-level diffing algorithm.
- `@types/diff`: TypeScript definitions for jsdiff.
- (Standard `yo office` dependencies): React, Fluent UI, Office.js types.

[Implementation Order]
Execute project scaffolding, followed by core services, UI, and finally integration testing.

1. Scaffold the project using `yo office` (React, TypeScript).
2. Clean up boilerplate UI and modify `manifest.xml` with appropriate naming and permissions.
3. Install `diff` and `@types/diff`.
4. Implement `llmService.ts` to handle generic OpenAI-compatible API calls.
5. Implement `diffService.ts` utilizing `diffWords`.
6. Implement `wordService.ts` to handle `ChangeTrackingMode.trackAll` and applying edits.
7. Build the `App.tsx` and `Settings.tsx` components using Fluent UI.
8. Wire the UI components to the services.
9. Perform end-to-end testing with sideloading in Word.
